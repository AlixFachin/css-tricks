Items in HTML are stacked above each other, from top to bottom. By default, between each element is inserted a new line.

Up until HTML5, elements could be divided into block-level elements and inline elements.
Inline elements cannot contain other elements.

This is an example of container

Each block-level element has a defined height/width, which is either set with its property or defaults to the size of its children elements.

Around each element content is the padding, then the border. Around the border is the margin until the next element.

You can see the padding, margin etc in Developer tools of your favourite browser.

Content height: 30px + Padding 10px x2 + Border 5px x2 = 60px box height

You might run to an issue when you want to define the size according to the viewport size or the parent size.

By default the height/weight of an item refers to the content dimensions, and not the total dimensions.

In order to apply the CSS properties to the total size (content+padding+border), we use the directive box-sizing: border-box;

Box width of 100% with default box-sizing.
Box width of 100% with border-box mode.
You can specify dimensions with several units. Here are the most useful:
  • px is the dimension (height/width) in pixels
  • % is the dimension in % of the parent element
  • vh,wh is in % of the viewport. (vh is for viewport height)
  • emis the font size at this place of the document
  • remis the root font size of the document
padding of 3em with big font size, padding of 57px
padding of 3rem with big font size, padding of 48px
padding of 3rem with small font size, padding of 48px

In order to make things even easier/more powerful, you can define variables in CSS. --variable-name: variable-value;. And to use the variable, you use var(--variable-name).

Where it becomes interesting is when you combine the use of variables and calculations. 


           --item-width: 150px;
           width: calc(max(40%, var(--item-min-width)));

I have no minimum width
I have a min width of 50 px
